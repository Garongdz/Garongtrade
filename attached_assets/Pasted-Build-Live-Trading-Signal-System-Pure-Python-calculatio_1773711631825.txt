Build Live Trading Signal System.
Pure Python calculations, minimal Claude API.

## API SOURCES
- Binance API: no key needed
- CoinGlass free tier: COINGLASS_API_KEY in secrets
- DeFiLlama: no key needed
- Mempool.space: no key needed
- CoinGecko: no key needed
- Blockchair: no key needed

## SCAN ENGINE
Interval options: 2h / 4h / 6h (default 2h)
No 1h option — minimum is 2h.
Coins: BTC, ETH, SOL, BNB, XRP, AVAX, ARB, OP, LINK, DOGE
CoinGlass data: only BTC, ETH, SOL, BNB, XRP
Others use Binance API for derivatives.
CoinGlass always fetches every 2h regardless 
of scan interval to stay under 10,000/month.

## ERROR HANDLING
If any API is down or times out:
- Set that layer score = 0 (neutral)
- Continue scan with available data
- Show warning banner on signal card:
  "⚠ [Nama API] sedang tidak tersedia, 
   data layer ini dilewati"
- Never crash the scan

## LAYER 1: TECHNICAL (max ±2)
Only 2 indicators: Support/Resistance + RSI
(small weight — lagging, easily manipulated)

RSI 14 on 4h (from Binance candles):
score_rsi = 0
if rsi < 30: score_rsi = +1
elif rsi > 70: score_rsi = -1

Support/Resistance on 4h:
Find key levels from last 20 candles:
resistance = max(high_candles_4h[-20:])
support = min(low_candles_4h[-20:])
mid = (resistance + support) / 2
price_position = (current_price - support) / 
                 (resistance - support)

score_sr = 0
if price_position < 0.25:
    score_sr = +1  (near support = potential bounce)
elif price_position > 0.75:
    score_sr = -1  (near resistance = potential rejection)

TECHNICAL TOTAL = score_rsi + score_sr
HARD CAP: max +2, min -2

## LAYER 2: DERIVATIVES (max ±5)
For BTC/ETH/SOL/BNB/XRP: CoinGlass free API
For others: Binance API

Funding Rate (most important):
score_funding = 0
if funding < -0.03%: score_funding = +2
elif funding < -0.01%: score_funding = +1
elif funding <= 0.01%: score_funding = 0
elif funding <= 0.03%: score_funding = -1
else: score_funding = -2

Open Interest 4h change:
score_oi = 0
if oi_change > +5% AND price_change > 0: score_oi = +1
elif oi_change > +5% AND price_change < 0: score_oi = -1
elif oi_change < -5%: score_oi = -1

Long/Short Ratio:
score_ls = 0
if ls_ratio < 0.85: score_ls = +1
elif ls_ratio > 1.5: score_ls = -1

Taker Buy/Sell Volume:
score_flow = 0
if buy_ratio > 0.55: score_flow = +1
elif buy_ratio < 0.45: score_flow = -1

DERIVATIVES TOTAL = score_funding + score_oi + 
                    score_ls + score_flow
HARD CAP: max +5, min -5

## LAYER 3: ON-CHAIN (max ±4)

Exchange Netflow proxy (Blockchair):
GET https://api.blockchair.com/bitcoin/stats
score_netflow = 0
if mempool_dropping AND tx_volume_falling: 
    score_netflow = +2
elif tx_volume_falling: 
    score_netflow = +1
elif tx_volume_rising_sharply: 
    score_netflow = -1
elif mempool_rising AND tx_volume_rising: 
    score_netflow = -2

Stablecoin Supply (DeFiLlama):
GET https://stablecoins.llama.fi/stablecoins
score_stable = 0
if usdt_usdc_supply_24h_change > +1%: score_stable = +1
elif usdt_usdc_supply_24h_change < -1%: score_stable = -1

DeFi TVL (DeFiLlama):
GET https://api.llama.fi/v2/globalCharts
score_tvl = 0
if tvl_24h_change > +3%: score_tvl = +1
elif tvl_24h_change < -3%: score_tvl = -1

BTC Mempool (Mempool.space):
GET https://mempool.space/api/mempool
score_mempool = 0
if mempool_count > 50000: score_mempool = +0.5
elif mempool_count < 5000: score_mempool = -0.5

For altcoins: use same BTC on-chain score 
as market-wide context.

ON-CHAIN TOTAL = score_netflow + score_stable + 
                 score_tvl + score_mempool
Round to nearest 0.5
HARD CAP: max +4, min -4

## LAYER 4: MACRO (max ±4)

Fear & Greed Index:
score_fg = 0
if fg < 20: score_fg = +2
elif fg < 35: score_fg = +1
elif fg > 80: score_fg = -2
elif fg > 65: score_fg = -1

BTC Dominance (CoinGecko):
score_dom = 0
For BTC:
  if dom_24h_change > +0.5%: score_dom = +1
  elif dom_24h_change < -0.5%: score_dom = -1
For altcoins:
  if dom_24h_change > +0.5%: score_dom = -1
  elif dom_24h_change < -0.5%: score_dom = +1

News Sentiment (from existing news feed):
score_news = 0
if bullish_count > bearish_count + 5: score_news = +1
elif bearish_count > bullish_count + 5: score_news = -1

MACRO TOTAL = score_fg + score_dom + score_news
HARD CAP: max +4, min -4

## GRAND TOTAL
raw_score = technical + derivatives + on_chain + macro
Max = 2+5+4+4 = 15, Min = -15
normalized = round((raw_score / 15) * 10, 1)
confidence = abs(normalized) * 10

## SIGNAL TRIGGER (Opsi B — include RISKY)
normalized >= +6.0 → LONG (confidence >= 60%)
normalized <= -6.0 → SHORT (confidence >= 60%)
normalized +5.0 to +5.9 → WEAK LONG → RISKY ⚠
normalized -5.0 to -5.9 → WEAK SHORT → RISKY ⚠
-4.9 to +4.9 → WAIT, no signal

## RISK CATEGORIES
confidence >= 80% = SAFE ✅ 
  max leverage 10x, recommended to trade
confidence 60-79% = MODERAT 🟡 
  max leverage 5x, trade with normal size
confidence 50-59% = RISKY 🟠 
  max leverage 2x, small size only
  show warning: "Setup lemah, hati-hati"

## ENTRY & LEVELS
entry_low  = current_price * 0.995
entry_high = current_price * 1.005

For LONG:
swing_low = min(low_candles_4h[-5:])
stop_loss = swing_low * 0.997
sl_distance = current_price - stop_loss
tp1 = current_price + (sl_distance * 1.5)
tp2 = current_price + (sl_distance * 3.0)
tp3 = current_price + (sl_distance * 5.0)

For SHORT:
swing_high = max(high_candles_4h[-5:])
stop_loss = swing_high * 1.003
sl_distance = stop_loss - current_price
tp1 = current_price - (sl_distance * 1.5)
tp2 = current_price - (sl_distance * 3.0)
tp3 = current_price - (sl_distance * 5.0)

## SIGNAL VALIDITY — 4 HOURS
Auto-expire signal if ANY condition met:
1. Signal age > 4 hours → EXPIRED ⏰
2. Price moves > 3% away from entry zone → EXPIRED ⏰
3. New scan produces opposite direction → EXPIRED ⏰

## DUPLICATE PREVENTION
Before creating signal, check database:
If ACTIVE signal exists for same coin 
AND same direction → SKIP
Only create new when previous is CLOSED.

## STATUS TRACKING (every 1 minute)
For LONG:
price >= tp1 → TP1_HIT ✅
price >= tp2 → TP2_HIT ✅✅
price >= tp3 → TP3_HIT ✅✅✅
price <= stop_loss → SL_HIT ❌
For SHORT: inverse

## WINRATE (closed signals only)
Exclude EXPIRED from winrate calculation.
win = any TP hit
loss = SL hit
winrate = wins / (wins + losses) * 100
profit_factor = total_profit_pct / total_loss_pct
